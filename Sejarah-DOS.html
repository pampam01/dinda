Sejarah Sistem Operasi DOS

  • Dinda Putri Ramadani (8020220216)
  • 07 Oktober 2023
  • Tugas 1Sejarah

MS-DOS, singkatan dari Microsoft Disk Operating System, adalah sebuah sistem operasi yang sangat banyak digunakan oleh komputer IBM-PC atau yang kompatibel dengannya. Microsoft membuat MS-DOS sebagai sebuah sistem operasi mainstream, sebelum pada akhirnya menghentikan dukungan MS-DOS secara perlahan ketika mereka membuat sebuah sistem operasi berbasis antarmuka grafis (dikenal juga dengan sebutan GUI) untuk pasar mainstream, yang disebut sebagai Microsoft Windows.

Disebut DOS (Disk Operating system) alasannya sistem operasi ini tersimpan dalam suatu disket maupun harddisk. MS-DOS bekerjsama dibentuk oleh sebuah perusahaan pembuat komputer berjulukan Seattle Computer Products (SCP) yang dikepalai oleh Tim Patterson yang direkrut oleh Microsoft untuk membuatkan DOS pada tahun 1980 sebagai sebuah perangkat lunak sistem operasi dengan nama Q-DOS (singkatan dari Quick and Dirty Operating System), yang selanjutnya diubah namanya menjadi 86-DOS,karena Q-DOS didesain semoga sanggup berjalan pada komputer dengan prosesor Intel 8086. Microsoftpun membeli lisensinya dengn harga 50.000 dolar Amerika dari SCP, kemudian mengubah namanya menjadi MS-DOS. Selanjutnya, ketika IBM hendak meluncurkan komputer pribadi yang disebut dengan IBM PC, Microsoft pun menjual lisensi MS-DOS kepada IBM.

Versi 1.00

Dibuat tanggal 12 Agustus 1981, Ini ialah versi awal di mana MS-DOS pertama kali diluncurkan oleh Microsoft. Isinya, tentu saja sangat primitif dan hanya berisi kerangka dasar saja: tidak ada tampilan grafik untuk mengelola berkas, tidak mendukung jaringan. Pokoknya, versi pertama DOS ini higienis dari embel-embel. Versi pertama dari DOS ini hanya mendukung IBM PC Model 5150 dan semua perangkat keras pendukungnya, ibarat hanya mendukung disket single-side floppy disk dengan ukuran 5½ inci berkapasitas 160K saja. EDLIN.COM; EXE2BIN.COM ; LINK.EXE ; DEBUG.COM ; CHKDSK.COM ; DISKCOMP.COM ; DISKCOPY.COM ; COMP.COM ; SYS.COM ; RAMCLEAR.COM ; SETCLOCK.COM ; MODE.COM ; FORMAT.COM ; BASIC.COM ; BASICA.COM ; LABEL.EXE.

Versi 1.1

Dibuat Mei 1982 Dukungan double side floppy disk 320K, dan file-file yang ada di dalamnya sama dengan versi 1.00, hal ini diakibatkan oleh IBM yang meluncurkan produk floppy disk drive gres untuk disket double-side dengan ukuran 5½ inci berkapasitas 320K. Penggunaan double-side bagi floppy disk sangatlah berpengaruh, alasannya media penyimpanan tersebut sanggup menyimpan dua kali lebih banyak dibandingkan dengan disket floppy single-side. Jangan tertawa melihat kapasitas disket yang didukung oleh MS-DOS 1.1.

Versi 1.25

Dibuat pada Mei 1982 Rilis pertama untuk klon IBM PC. Dan alasannya itulah IBM PC dan klon IBM PC yang kompatibel dengan IBM PC sangat sukses di pasaran. Ini merupakan kesuksesan tiga perusahaan: IBM, Intel, dan tentu saja Microsoft. File yang ada di dalam berkembang atau bertambah dari versi 1.00 : TREE.COM ; MKDIR/MD ; CHDIR/CD ; RMDIR/RD ; MORE.COM ; LESS.COM ; SORT.COM ; ASSIGN.COM ; COLOR.COM

Versi 2.00

Dibuat pada Maret 1983 Dukungan untuk floppy disk 360K. derma untuk penggunaan struktur direktori yang hierarkis (bercabang) ibarat layaknya sebuah pohon yang mempunyai akar (root) dan cabang (branch). Selain itu, MS-DOS 2.0 juga mendukung banyak sekali macam fungsi yang diadopsi dari UNIX, ibarat halnya penggunaan I/O redirection (>, >>, <, <<) dan print spooler. Fitur print spooler ialah fitur yang sanggup menyimpan berkas yang akan dicetak dalam sebuah memori penyangga sehingga berkas yang akan dicetak tidak pribadi dicetak, melainkan disimpan terlebih dahulu.

Tujuannya ialah semoga proses pencetakan sanggup berlangsung secara lebih cepat, dan pengguna tidak harus direpotkan untuk mencetak keseluruhan berkas kalau kertasnya habis, melainkan hanya membutuhkan kertas gres dan menekan tombol resume untuk melanjutkan. Sedangkan I/O redirection berfungsi untuk melaksanakan fungsi input dan output yang dilakukan dari baris perintah. Selain kedua fitur yang diadopsi dari UNIX, DOS versi 2.0 juga memperlihatkan keleluasaan kepada Anda semoga sanggup memasang driver untuk DOS, dengan mengintegrasikannya di dalam berkas konfigurasi CONFIG.SYS.GRAPHICS.COM ; GRAFTABL.COM ; PARK.COM ; PRINT.COM ; BACKUP.EXE ; RESTORE.EXE

Versi 2.05

Dibuat pada April 1983, Dukungan abjad kanji (Jepang). Jenis Filenya sama dengan versi 2.0. Mendukung time, date, currency, dan decimal symbols, 16-bit Japanese Kanji.Tahun 1984, IBM menciptakan PC AT dengan prosesor Intel 80286 yang menyediakan pengalamatan extended (extended addressing) dan karakteristik perlindungan memori (memory protection), user dan kernel mode, ring-based protection dan kemampuan menjalankan banyak aktivitas sekaligus. PC AT menyediakan disket 1.2M, battery backup clock, gosip konfigurasi pada CMOS, harddisk lebih dari 10M, mendukung RAM disk.

Versi 2.10

Dibuat pada Okotber 1983, tidak ada perubahan yang signifikan pada versi ini jenis filenya pun masih sama.

Versi 2.11

Dibuat pada Maret 1984, juga tidak ditemukan perkembangan yang terbaru sama ibarat versi 2.10.

Versi 3.00

Dibuat pada Agustus 1984, DOS 3.0 ialah versi yang sanggup mendukung fungsi jaringan, walaupun terbatas pada jaringan dengan konsep kelompok kerja (workgroup) dan terkoneksi sebagai sebuah klien pada sebuah server. Pengguna sanggup memakai teknologi jaringan yang dibentuk oleh IBM, Token Ring pada sistem operasi DOS. DOS 3.0 juga mendukung floppy disk 3½ inci low-density yang berkapasitas 720K, Dengan penambahan file FDISK.EXE ; FC.EXE ; APEND.COM ; RECOVER.COM ; NLSFUNC.EXE.

Versi 3.10

Dibuat pada November 1984, Mendukung jaringan lokal (LAN) dengan komponen MS-NET, serta penambahan beberapa file ibarat :SUBST.EXE ; SELECT.COM ; FIND.EXE ; SHARE.EXE ; REPLACE.EXE

Versi 3.20

Dibuat pada Januari 1986. Dukungan untuk topologi jaringan IBM Token Ring, juga derma untuk Flopy disk 720K 3.5 inci dengan penambahan file Xcopy.exe.

Versi 3.30

Dibuat pada April 1987, DOS 3.3 mendukung floppy disk 3½ inci high-density yang berkapasitas 1.44M. Dalam DOS versi 3 juga sanggup mendukung partisi pada hard disk yang mempunyai kapasitas hingga 32M, 12M lebih tinggi dibandingkan dengan DOS versi 2.0., dengan penambahan file :FASTOPEN.EXE ; FASTHELP.EXE

Versi 3.40

Dibuat pada Juni 1988, sistem operasi dengan tampilan grafis mulai menjamur. Microsoft juga meluncurkan Windows versi 1.0 pada tahun 1985 dan Windows versi 2.0 pada tahun 1989. Pada DOS versi 4.0 ini, Microsoft menciptakan sebuah aktivitas shell dengan nama DOS Shell, yang mempunyai tampilan ibarat DOS Executive milik Windows versi 1.0. Pada ketika itu, ada sebuah aktivitas shell yang juga terkenal yang berjalan di atas sistem operasi DOS, dengan nama Norton Commander (NC) yang dibentuk oleh Peter Norton.

Dengan memakai shell, pengguna sanggup lebih gampang mengorganisasikan berkas dan arsip, dan menjalankan program, dalam satu layar saja. Selain itu, alasannya mouse sanggup mempermudah pengoperasian komputer, DOS Shell juga sanggup mendukungnya, dan penambahan beberapa file ibarat : MOUSE.COM ; HIMEM.SYS ; SMARTDRV.EXE ; SETVER.EXE ; MIRROR.COM

Versi 4.01

Dibuat pada November 1988, hanya penambahan file RAMDRIVE.SYS

Versi 5.0

Dibuat pada Juni 1991,IBM memang gagal memperpanjang umur IBM PC, akan tetapi Microsoft terus membuatkan DOS dengan meluncurkan MS-DOS versi 5.0. Versi 5.0 ini dirilis pada tahun 1991, dan mempunyai banyak sekali fitur yang jauh lebih baik, ibarat halnya device driver yang sanggup diinstal pada area memori di atas 640K (HMA), dengan memakai parameter DEVICEHIGH pada berkas konfigurasi CONFIG.SYS dan LOADHIGH pada berkas AUTOEXEC.BAT sehingga memperbanyak kemungkinan untuk menjalankan aktivitas yang membutuhkan conventional memory (memori 640K awal yang dimiliki oleh komputer) yang lebih besar. dengan penambahan file : EDIT.COM ; QBASIC.EXE ; DOSSHELL.EXE ; DOSKEY.EXE ; UNDELETE.EXE ; UNFORMAT.EXE ; MWUNDEL.EXE ; EXPAND.EXE

Versi 6.0

Dibuat pada Maret 1993, Versi ini mempunyai banyak utilitas tambahan, ibarat halnya Antivirus (MSAV.EXE), Virus Protection (VSAFE.COM) yang diadopsi dari Central Point PC-Tools. Selain itu, ada juga utilitas untuk melaksanakan defragmentasi hard disk (DEFRAG.EXE) yang diadopsi dari Symantec Norton Utilities, utilitas untuk mengusut hard disk dari kesalahan penempatan berkas dan kerusakan (SCANDISK.EXE) sebagai pengganti CHKDSK.EXE, dan banyak sekali perbaikan dari utilitas yang ada sebelumnya, dengan penambahan beberapa file ibarat : EMM386.EXE ; DBLSPACE.EXE ; VSAFE.COM ; MSAV.EXE ; DEFRAG.EXE ; SCANDISK.EXE

Versi 6.20

Dibuat pada November 1993, mengurangi file DBLSPACE.exe, alasannya bermasalah dengan hak paten dengan Stac Techology sebagai pemilik paten file tersebut.

Versi 6.22

Dibuat pada Juni 1994, file DBLSPACE.EXE diganti DRVSPACE.EXE dengan fungsi yang sama, merupakan versi terakhir dari DOS yang bersifat stand-alone.

Versi 7.0

Dibuat pada April 1995, dirilis hanya beberapa bulan sebelum Microsoft meluncurkan Windows 95, sebuah versi Windows yang benar-benar independen dari MS-DOS. Sebenarnya, Windows 95 tidaklah benar-benar 100% bebas dari MS-DOS, hanya saja memang beberapa fungsionalitas yang sebelumnya ditangani oleh kernel MS-DOS, kini ditangani oleh Windows. Berkas MSDOS.SYS yang pada versi-versi sebelumnya ialah berkas biner (program), kini hanya berkas teks biasa, yang berisi perihal proses booting komputer saja, dan banyak sekali macam baris berisi “xxx” dengan fungsi yang tidak jelas.

Versi 7.10

Dibuat pada Agustus 1997, Ketika Microsoft meluncurkan Windows 98, Microsoft juga merevisi DOS menjadi versi 7.10 dengan derma untuk sistem berkas FAT32 yang jauh lebih efisien daripada sistem berkas FAT16. Fungsionalitas yang dimiliki MS-DOS 7.10 pun banyak dipangkas. Hasilnya, pada selesai tahun 1999, Microsoft meluncurkan versi Windows baru, dengan nama arahan Georgia (Windows Millennium Edition) yang benar-benar tanpa DOS.

Sebenarnya, sistem operasi yang benar-benar tanpa DOS pernah dibentuk oleh Microsoft sebelumnya dengan Windows NT, hanya saja Windows NT ditujukan untuk kalangan minicomputer, server jaringan, mainframe, dan juga workstation. Penggunaannya yang rumit menjadikannya sangat sulit diadopsi oleh para pemula di bidang komputer.
